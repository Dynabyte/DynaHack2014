DynaHack 2014

Välkommen till DynaHack, en lagtävling där ni bygger en spelmotor för att tävla i det klassiska spelet Sänka skepp!

Tävlingen

Tävlingen går ut på att som ett lag gemensamt bygga en REST-server som ska svara på ett förutbestämt API. Ni kommer få POST-anrop skickade mot servern innehållandes den aktuella spelplanen. Er uppgift är att bygga en algoritm för att välja vilken position på motståndarens spelplan ni vill skjuta på.

När ni får spelplanen postad till er första gången kommer det precis som i det riktiga spelet bara vara okända positioner, men för varje position ni skjuter på kommer nästa spelplan ni får postad till er att ha mer information.

Under själva tävlingsmomentet kommer vi att generera fram en spelplan med skepp. Alla lag får samma spelplan postad till sig och det gäller att sänka skeppen med minst antal skott!

Regler

I vår version av Sänka skepp så gäller följande regler:

  • Det är tillåtet att skjuta utanför spelplanen (men räknas som "miss")
  • Det är tillåtet att skjuta på en position som man redan skjutit på (men räknas som "miss")
  • Spelplanen kan variera i storlek
  • Antalet skepp kan variera i antal och storlek

Protokollet

Er server kommer att få en JSON postad till sig som innehåller all info ni behöver för att välja ert nästa drag.

Schema

09.00 - Intro

10.30 - Första deltävling!

12.00 - Lunch

14.30 - Andra deltävling!

16.30 - Final!

17.00 - Mingel

Simulering

För att kunna få en känsla av hur ens spelmotor presterar så har vi snickrat ihop ett litet program för att testa spela upp en hel spelomgång.

Programmet kräver Python (2.7) och startas med: python tester.py URL.

Vill man stress-testa sitt program och få veta sitt medelvärde så kan man använda: python tester.py URL stress.

Färdiga projekt

För att snabbt komma igång har vi satt upp några enkla projekt i olika språk. Med dem slipper du sätta upp REST-servern och kan börja direkt på din spelmotor.

De olika spelmotorerna är gjorda så de svarar med en giltigt, men helt randomiserad position.

Projekten är zip:ade och innehåller mer beskrivningar om hur man sätter upp dem.

  • Java 7
  • Node.js
  • Python
  • C#

De flesta hjälp-filer i projekten är skrivna i Markdown, vilket kan läsas enkelt med till exempel Texts.
